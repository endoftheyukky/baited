# baited

体験型インスタレーション — 走光性とアテンションエコノミー

鑑賞者が懐中電灯型デバイスで光を照射すると、ディスプレイ上の虫の群れが光源に集まり、やがて飽きて離散する。人々の注意と関心が資源として抽出される「アテンションエコノミー」の構造を、虫の走光性に翻訳した作品。

虫の動きは、対数螺旋（光コンパス反応）、レヴィフライト、背光反応、間欠運動、馴化といった実在の昆虫行動研究に基づくアルゴリズムで生成されている。

## セットアップ

### 前提
- Node.js v18以上（https://nodejs.org/ja）

### インストール
```bash
cd baited
npm install
```

### メディアファイルの配置
```
main/audio/
  baited.wav               ← アンビエントBGM

sub-display/videos/
  video-1.mp4              ← サブディスプレイ映像（ピエロ、ミュート再生）
```

### (推奨) p5.js をオフライン用に保存

展示会場にインターネットがない場合に必要。

1. https://cdnjs.cloudflare.com/ajax/libs/p5.js/1.9.4/p5.min.js をダウンロード
2. `main/` フォルダに保存
3. `main/index.html` の CDN パスを `<script src="p5.min.js">` に変更

## 展示当日の起動手順

### Step 1: ネットワーク接続

1. Wi-Fiルーターの電源を入れる
2. メインPCをそのWi-Fiに接続する
3. iPadも同じWi-Fiに接続する

### Step 2: メインPCのIPアドレスを確認

1. コマンドプロンプトを開く（Windowsキー → `cmd` と入力 → Enter）
2. 以下を入力して Enter:
```
   ipconfig
```
3. 「Wi-Fi」のセクションから **IPv4 アドレス** をメモする
```
   Wireless LAN adapter Wi-Fi:
      IPv4 Address. . . . . . . : 192.168.1.5    ← これをメモ
```

### Step 3: サーバーを起動
```bash
cd baited
npm start
```

以下の表示が出れば成功:
```
=== baited server ===
  Main:      http://localhost:3000
  Sub:       http://localhost:3000/sub?id=1
  Status:    http://localhost:3000/status
  Analytics: http://localhost:3000/analytics
  WS:        ws://localhost:3001
```

> ⚠️ 初回起動時に「ファイアウォールでブロックされました」というダイアログが出たら、**「アクセスを許可する」を必ず押す**。これを許可しないとiPadから繋がらない。

### Step 4: メイン映像を開く

メインPCのブラウザ（Chrome推奨）で:
```
http://localhost:3000
```

- 黒い画面に「Click to start」と表示される
- クリックするとシミュレーションとBGMが開始される
- 外部モニターにブラウザを移動し、**F11** で全画面表示

### Step 5: iPadでサブ映像を開く

iPadのSafariで（`192.168.1.5` の部分はStep 2のIPアドレスに置き換え）:
```
http://192.168.1.5:3000/sub?id=1
```

映像はミュートなので、タップ不要で自動的に待機状態になる。光のON/OFFに連動してピエロ映像がフェードイン/アウトする。

> 💡 Safariで「ホーム画面に追加」するとアドレスバーなしの全画面表示になる。
> 💡 iPadの「設定 → 画面表示と明るさ → 自動ロック」を「なし」に設定しておく。

### Step 6: Wii Remote を接続

1. Wiimoteドライバーを起動
2. Wii RemoteのBluetoothペアリングを実行
3. マウスとして動作すれば準備完了

### Step 7: 動作確認

- メイン画面でマウスを押しっぱなし → 虫が光に集まる
- iPadにピエロ映像が表示される
- スピーカー（メインモニター内蔵）からBGMが聞こえる
- 離すと全部止まる

`main/index.html` をブラウザで直接開く。虫のシミュレーションとBGMだけ動く（iPad連携なし）。

## 展示中のモニタリング

同じWi-Fiに繋がったスマホから:

| URL | 内容 |
|-----|------|
| `<IP>:3000/status` | 接続中の端末一覧（5秒自動更新） |
| `<IP>:3000/analytics` | 全日程の体験データ概要 |

## パラメータ調整

実行中に **D キー** でパラメータパネルを表示。値を変更するとリアルタイムに反映され、ブラウザに自動保存される。PC再起動後も設定は保持される。**R キー** でデフォルトに戻る。

| キー | 機能 |
|------|------|
| **D** | パネル表示/非表示 |
| **↑↓** | パラメータ選択 |
| **←→** | 値を増減 |
| **R** | デフォルトに戻す |

| パラメータ | 初期値 | 説明 |
|-----------|--------|------|
| `numBugs` | 250 | 虫の数（PCの性能に応じて調整） |
| `lightRadius` | 220 | 光の円の半径（モニターの大きさに応じて調整） |
| `bugSizeMin` | 14 | 虫の最小サイズ（モニターからの距離で調整） |
| `bugSizeMax` | 24 | 虫の最大サイズ |

## トラブルシューティング

**iPadから繋がらない**
→ メインPCとiPadが同じWi-Fiに繋がっているか確認
→ IPアドレスが合っているか確認（`ipconfig` でもう一度確認）
→ ファイアウォールで Node.js が許可されているか確認

**iPadの映像が再生されない**
→ Safariをリロード
→ `sub-display/videos/video-1.mp4` が配置されているか確認

**BGMが鳴らない**
→ メイン画面で「Click to start」を押したか確認
→ PCの音量・モニターの音量がミュートになっていないか確認
→ `main/audio/baited.wav` が配置されているか確認

**虫の動きがカクカクする**
→ D キーでパネルを開き FPS を確認
→ `numBugs` を 200 や 150 に下げてみる

**パラメータをリセットしたい**
→ D キーでパネルを開き R キーを押す
→ ブラウザの保存データも含めてデフォルトに戻る

**サーバーが起動しない**
→ Node.js がインストールされているか確認（`node -v`）
→ `npm install` を実行したか確認
