# baited

体験型インスタレーション — 走光性とアテンションエコノミー

鑑賞者が懐中電灯型デバイスで光を照射すると、ディスプレイ上の虫の群れが光源に集まり、やがて飽きて離散する。人々の注意と関心が資源として抽出される「アテンションエコノミー」の構造を、虫の走光性に翻訳した作品。

虫の動きは、対数螺旋（光コンパス反応）、レヴィフライト、ラン・アンド・タンブル、馴化といった実在の昆虫行動研究に基づくアルゴリズムで生成されている。

---

## 必要なもの

- メインPC（Windows、Node.js v18以上）
- メインモニター（HDMI接続）
- スピーカー（メインPCの音声出力）
- iPad × 3
- Wi-Fiルーター 1台（自前持ち込み推奨）
- Wii Remote + センサーバー + Wiimoteドライバー

## 初回セットアップ

### 1. Node.js をインストール

https://nodejs.org/ から LTS版をダウンロードしてインストール。

### 2. 依存パッケージをインストール
```bash
cd baited
npm install
```

### 3. メディアファイルを配置
```
main/audio/
  ambient.mp3         ← アンビエントBGM

sub-display/videos/
  video-1.mp4         ← iPad 1 の映像（音声含む）
  video-2.mp4         ← iPad 2 の映像（音声含む）
  video-3.mp4         ← iPad 3 の映像（音声含む）
```

### 4. (推奨) p5.js をオフライン用に保存

展示会場にインターネットがない場合に必要。

1. https://cdnjs.cloudflare.com/ajax/libs/p5.js/1.9.4/p5.min.js をダウンロード
2. `main/` フォルダに保存
3. `main/index.html` を開き、以下を変更:
```html
   <!-- 変更前 -->
   <script src="https://cdnjs.cloudflare.com/ajax/libs/p5.js/1.9.4/p5.min.js"></script>
   <!-- 変更後 -->
   <script src="p5.min.js"></script>
```

## 展示当日の起動手順

誰でも起動できるように、上から順番にやれば動きます。

### Step 1: ネットワーク接続

1. Wi-Fiルーターの電源を入れる
2. メインPCをそのWi-Fiに接続する
3. iPad 3台もすべて同じWi-Fiに接続する

### Step 2: メインPCのIPアドレスを確認

1. メインPCでコマンドプロンプトを開く（Windowsキー → `cmd` と入力 → Enter）
2. 以下を入力して Enter:
```
   ipconfig
```
3. 表示の中から「Wi-Fi」のセクションを探し、**IPv4 アドレス** をメモする
```
   Wireless LAN adapter Wi-Fi:
      IPv4 Address. . . . . . . : 192.168.1.5    ← これをメモ
```

### Step 3: サーバーを起動

コマンドプロンプトで:
```bash
cd baited
npm start
```

以下のような表示が出れば成功:
```
=== baited server ===
  Main:    http://localhost:3000
  Sub:     http://localhost:3000/sub?id=1
  Status:  http://localhost:3000/status
  WS:      ws://localhost:3001
```

> ⚠️ 初回起動時に「ファイアウォールでブロックされました」というダイアログが出たら、**「アクセスを許可する」を必ず押してください**。これを許可しないとiPadから繋がりません。

### Step 4: メイン映像を開く

メインPCのブラウザ（Chrome推奨）で以下を開く:
```
http://localhost:3000
```

- 黒い画面に「Click to start」と表示される
- クリックするとシミュレーションとBGMが開始される
- 外部モニターにブラウザを移動し、**F11** で全画面表示

### Step 5: iPadでサブ映像を開く

各iPadのSafariで以下を開く（`192.168.1.5` の部分はStep 2でメモしたIPアドレスに置き換え）:
```
iPad 1:  http://192.168.1.5:3000/sub?id=1
iPad 2:  http://192.168.1.5:3000/sub?id=2
iPad 3:  http://192.168.1.5:3000/sub?id=3
```

- 黒い画面に「Tap to start」と表示される
- 1回タップすると待機状態になる（以降は自動で連動）

> 💡 iPadのSafariで「ホーム画面に追加」すると、アドレスバーなしの全画面で表示できます。

> 💡 iPadの「設定 → 画面表示と明るさ → 自動ロック」を「なし」にしておくと、展示中にスリープしません。

### Step 6: Wii Remote を接続

1. Wiimoteドライバーを起動
2. Wii RemoteのBluetoothペアリングを実行
3. マウスとして動作すれば準備完了

### Step 7: 動作確認

- メイン画面でマウスクリック（押しっぱなし）→ 虫が光に集まる
- iPad 3台に映像が表示される
- スピーカーからBGMが聞こえる
- 離すと全部止まる

## トラブルシューティング

**iPadから繋がらない**
→ メインPCとiPadが同じWi-Fiに繋がっているか確認
→ IPアドレスが合っているか確認（`ipconfig` でもう一度確認）
→ ファイアウォールで Node.js が許可されているか確認

**iPadの映像が再生されない**
→ 「Tap to start」をもう一度タップ
→ Safariをリロード（下にスワイプ）

**BGMが鳴らない**
→ メイン画面で「Click to start」を押したか確認
→ PCの音量がミュートになっていないか確認
→ `main/audio/ambient.mp3` が配置されているか確認

**虫の動きがカクカクする**
→ D キーでパネルを開き、FPSを確認
→ `numBugs` を 200 や 150 に下げてみる

**サーバーが起動しない**
→ Node.js がインストールされているか確認（`node -v` で確認）
→ `npm install` を実行したか確認

## パラメータ調整

実行中に **D キー** でパラメータパネルを表示。リアルタイムに調整できる。

| キー | 機能 |
|------|------|
| **D** | パネル表示/非表示 |
| **↑↓** | パラメータ選択 |
| **←→** | 値を増減 |
| **R** | デフォルトに戻す |

| パラメータ | 初期値 | 説明 |
|-----------|--------|------|
| `numBugs` | 250 | 虫の数 |
| `lightRadius` | 220 | 光の円の半径 (px) |
| `reactionDelayMax` | 300 | 光ON後の反応遅延 (0〜この値フレーム) |
| `decayRateMin/Max` | 0.0004 / 0.0012 | 飽き速度 (小さい=遅い) |
| `recoveryRate` | 0.006 | 光OFF時の興味回復速度 |
| `boredomThreshold` | 0.15 | この値以下で飽き判定 |
